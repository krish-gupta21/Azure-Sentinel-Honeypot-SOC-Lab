# Threat Hunting and Analysis

## Objective

Analyze Windows Security Events collected by Microsoft Sentinel to identify authentication activity, detect unauthorized login attempts, and evaluate the effectiveness of the deployed detection rule.

---

## Investigation Overview

After configuring Microsoft Sentinel and enabling Windows Security Event collection, several Kusto Query Language (KQL) queries were executed to investigate authentication events generated by the Azure virtual machine. The collected telemetry was used to distinguish legitimate administrator logins from external unauthorized access attempts.

---

# Query 1 - Failed Authentication Attempts

### KQL Query

```kusto
SecurityEvent
| where EventID == 4625
| project TimeGenerated, Computer, Account, IpAddress, FailureReason, Status, SubStatus
| order by TimeGenerated desc
```

### Purpose

Retrieve all failed Windows authentication attempts.

### Observation

- Multiple failed authentication attempts (Event ID **4625**) were detected.
- External IP addresses attempted to authenticate against the publicly exposed RDP service.
- The targeted usernames included **Test** and **win11**.
- Status code **0xc000006d** indicates a logon failure.
- SubStatus **0xc0000064** indicates that the specified user account does not exist.

# Query 2 - Successful Authentication Events

### KQL Query

```kusto
SecurityEvent
| where EventID == 4624
| project TimeGenerated, Account, IpAddress, LogonTypeName
| order by TimeGenerated desc
```

### Purpose

Retrieve successful Windows authentication events.

### Observation

- Successful authentication events (Event ID **4624**) were recorded.
- The administrator account **Win11\krish** successfully connected using **Remote Desktop (Logon Type 10)**.
- Additional events generated by **NT AUTHORITY\SYSTEM** represent expected operating system activity.

# Query 3 - Top Source IP Addresses

### KQL Query

```kusto
SecurityEvent
| where EventID == 4625
| summarize Attempts=count() by IpAddress
| order by Attempts desc
```

### Purpose

Identify the external IP addresses responsible for the highest number of failed login attempts.

### Observation

| IP Address | Failed Attempts |
|------------|----------------:|
| 45.142.193.166 | 10 |
| 80.94.95.83 | 2 |
| 45.142.193.145 | 2 |

The IP address **45.142.193.166** generated the highest number of failed authentication attempts during the observation period.

# Query 4 - Targeted Usernames

### KQL Query

```kusto
SecurityEvent
| where EventID == 4625
| summarize Attempts=count() by TargetUserName
```

### Purpose

Determine which user accounts were targeted during failed authentication attempts.

### Observation

| Username | Attempts |
|----------|----------:|
| win11 | 10 |
| Test | 4 |

The attacker attempted authentication using multiple usernames, with **win11** being the primary target.

# Query 5 - Logon Types

### KQL Query

```kusto
SecurityEvent
| summarize Count=count() by LogonTypeName
```

### Purpose

Analyze the different Windows logon types observed in the collected security events.

### Observation

The dataset contains a combination of:

- **Logon Type 10 (RemoteInteractive)** – Remote Desktop sessions.
- **Logon Type 3 (Network)** – Network authentication events.
- **Logon Type 5 (Service)** – Windows service logons.
- **Logon Type 2 (Interactive)** – Local interactive logons.

Most events were generated by normal Windows background processes, while RemoteInteractive logons represent actual RDP sessions.

# Key Findings

- Microsoft Sentinel successfully ingested Windows Security Events from the Azure virtual machine.
- The deployed KQL queries identified both successful and failed authentication events.
- External systems attempted to authenticate against the exposed RDP service.
- The source IP **45.142.193.166** generated the highest number of failed login attempts.
- Multiple usernames were targeted during authentication attempts, indicating automated account enumeration.
- Authorized Remote Desktop access by the administrator account was successfully distinguished from unauthorized login attempts.
- The scheduled analytics rule can continuously monitor similar authentication activity and generate alerts for further investigation.

---

# Conclusion

The Microsoft Sentinel deployment successfully collected and analyzed Windows Security Events from the Azure virtual machine. Using KQL-based threat hunting, unauthorized authentication attempts originating from external IP addresses were identified while legitimate administrator activity was verified. This demonstrates how Microsoft Sentinel can be used to monitor exposed systems, investigate authentication events, and support security operations through centralized log analysis and automated detection.
